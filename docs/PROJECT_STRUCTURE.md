# Project Structure

This document provides an overview of the file and directory structure of the **MobileDear** project.

## Root Directory

- **.expo/**: Expo configuration and cache files.
- **android/**: Android native project files (generated by prebuild).
- **app/**: Main application source code (Expo Router).
- **assets/**: Static assets like images, fonts, and icons.
- **components/**: Reusable React components.
- **constants/**: Global constants (colors, layout, strings).
- **hooks/**: Custom React hooks.
- **mocks/**: Mock data for development and testing.
- **src/**: Core source code including services and utilities.
- **store/**: State management (Zustand) stores.
- **types/**: TypeScript type definitions.
- **utils/**: Utility functions and helpers.
- **docs/**: Project documentation.

## Key Directories Detail

### `app/` (Expo Router)

This directory follows the **Expo Router** file-based routing convention.

- `index.tsx`: The entry point (Home screen).
- `_layout.tsx`: Root layout configuration.
- `(tabs)/`: Tab-based navigation group.
- `admin/`: Admin panel routes and screens.
  - `dashboard.tsx`: Admin dashboard.
  - `products.tsx`: Product management.
  - `users.tsx`: User management.
  - `settings/`: App configuration settings.

### `src/`

- **services/**: API integration modules (Auth, Products, Settings, etc.).
- **utils/**: Helper functions (Alerts, Formatters).

### `store/`

Contains global state management using **Zustand**.

- `user-store.ts`: User authentication and profile state.
- `cart-store.ts`: Shopping cart state.

### `components/`

- **ui/**: Basic UI building blocks (Buttons, Inputs, Cards).
- **common/**: Shared components used across multiple screens.

## Naming Convention

- **Files**: `camelCase` or `kebab-case` (mostly `camelCase` for Utils/Hooks, `PascalCase` for Components).
- **Directories**: `lowercase` or `camelCase`.
- **Components**: `PascalCase`.
